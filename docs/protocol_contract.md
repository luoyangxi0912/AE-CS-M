# AE-CS-M Protocol Contract v2

## 1. Unified External Interface

Every public method must expose:

```python
fit(train_source, scaler=None, smoke=False, metadata=None)
impute(X, mask_observed, metadata=None)
```

Parameter meaning:

- `train_source`: training records or training-record collection
- `scaler`: scaling object for normalization and inverse normalization
- `smoke`: whether to run under Phase 1 smoke rules
- `metadata`: optional extra context such as `seed`, `file_id`, `train_files`

## 2. Unified Mask Semantics

The only public mask semantics in `AE-CS-M` are:

- `mask_observed = 1` means observed
- `mask_observed = 0` means missing
- `mask_missing = 1 - mask_observed`

Mentor missing-indicator semantics must not appear as experiment-layer public contract.

## 3. AE-CS vs Baseline Layering

### 3.1 AE-CS Body

AE-CS body directly uses:

- multivariate temporal tensor input
- explicit `mask_observed`
- inference-time observed-value preservation

### 3.2 Baseline Adapter Layer

Baseline adapters may internally:

- convert temporal windows to flatten/vector
- convert `mask_observed` to `mask_missing`
- preserve mentor-style internal training semantics

Baseline adapters may not:

- rewrite public interface semantics
- leak missing-indicator semantics into experiment public protocol

## 4. Data Protocol

Phase 1 supports only:

- single dataset: `battery_pack_wltp_dataset`
- single experiment shell: `experiments/battery_pack_wltp/`

Default assumptions:

- training source comes from complete sequences
- evaluation missingness is generated by experiment-level mask protocol

## 5. Result Artifact Protocol

Artifact directories:

- `experiments/battery_pack_wltp/results/raw/`
- `experiments/battery_pack_wltp/results/summary/`
- `experiments/battery_pack_wltp/masks/`
- `experiments/battery_pack_wltp/checkpoints/`

Artifact file naming:

- raw: `<file_id>__<config_name>__seed<seed>.json`
- mask: `<file_id>__<config_name>__seed<seed>.npz`
- summary: `by_seed.csv`, `config_summary.csv`, `main_table.csv`

## 6. Smoke Protocol

Phase 1 smoke is layered:

- `Phase 1A`: skeleton smoke
- `Phase 1B`: AE-family smoke
- `Phase 1C`: GAIN smoke

Recommended fixed smoke dimensions:

- single dataset
- single config
- single seed
- small record subset

## 7. Phase 1 Public Method Set

Phase 1 public methods:

- `aecs`
- `deep_ae`
- `sm_dae`
- `sdai`
- `trdae`
- `gain`

Rules:

- use stable public method names
- do not use mentor `model_id` as public identifier
- family internals may be shared, but registration stays method-based
- `gain` belongs only to `Phase 1C`

## 8. GAIN Adapter Protocol

`GAIN` is part of Phase 1, but only in `Phase 1C`.

Its external interface must still be:

```python
fit(train_source, scaler=None, smoke=False, metadata=None)
impute(X, mask_observed, metadata=None)
```

Its external mask contract must still be:

- `mask_observed = 1` means observed
- `mask_observed = 0` means missing

Inside the adapter, `GAIN` may:

- convert `mask_observed` to `mask_missing`
- preserve mentor missing-indicator training semantics
- preserve mentor generator/discriminator internal training flow

Inside the adapter, `GAIN` may not:

- expose mentor public interface semantics to the experiment layer
- require experiment-layer users to speak `mask_missing = 1`
- expose mentor `model_id`, old entry scripts, or old dataset objects as public protocol

Conclusion:

- `GAIN` may preserve mentor internals
- `GAIN` must behave externally as a standard AE-CS-M adapter
